# 设计原则

JianghuJS UI 组件库遵循以下设计原则，确保组件的一致性、可用性和可维护性。

## 一致性 (Consistency)

### 视觉一致性
- 所有组件遵循 Material Design 规范
- 统一的颜色系统和间距规范
- 一致的交互反馈和动画效果

### 代码一致性
- 统一的 props 命名规范
- 一致的事件命名和触发方式
- 标准化的插槽设计

## 简洁性 (Simplicity)

### 易于使用
- 提供合理的默认值，减少必填配置
- 清晰的 API 设计，避免复杂的嵌套结构
- 完善的文档和示例代码

### 代码简洁
- 组件职责单一，避免过度封装
- 保持组件的可组合性
- 减少不必要的依赖

## 灵活性 (Flexibility)

### 高度可定制
- 丰富的 props 支持各种使用场景
- 插槽机制支持自定义内容
- 支持主题定制和样式覆盖

### 可扩展性
- 组件设计预留扩展空间
- 支持事件监听和自定义行为
- 易于集成第三方功能

## 性能 (Performance)

### 优化策略
- 按需加载，减少初始包体积
- 合理使用计算属性和侦听器
- 避免不必要的重渲染

### CDN 优势
- 利用浏览器缓存
- 减少项目构建时间
- 降低部署复杂度

## 可访问性 (Accessibility)

### 无障碍设计
- 支持键盘导航
- 合理的 ARIA 标签
- 良好的色彩对比度

### 响应式设计
- 移动端优先
- 适配各种屏幕尺寸
- 支持触摸和鼠标交互

## 组件设计规范

### Props 设计

```javascript
// ✅ 好的设计
{
  color: 'primary',        // 使用预定义值
  size: 'medium',          // 语义化的尺寸
  disabled: false,         // 布尔值使用 is/has/should 等前缀或直接描述状态
}

// ❌ 避免的设计
{
  col: '#1976D2',          // 避免缩写
  s: 2,                    // 不清晰的命名
  dis: 0,                  // 用数字表示布尔值
}
```

### 事件设计

```javascript
// ✅ 好的设计
this.$emit('click', event);
this.$emit('change', value);
this.$emit('submit', formData);

// ❌ 避免的设计
this.$emit('c', event);              // 避免缩写
this.$emit('on-click', event);       // 不需要 on- 前缀
this.$emit('buttonClick', event);    // 避免组件名前缀
```

### 插槽设计

```vue
<!-- ✅ 好的设计 -->
<template>
  <div class="component">
    <slot name="header"></slot>
    <slot></slot>  <!-- 默认插槽 -->
    <slot name="footer"></slot>
  </div>
</template>

<!-- 提供作用域插槽支持自定义 -->
<template v-slot:item="{ item }">
  <slot name="item" :item="item">
    {{ item.name }}
  </slot>
</template>
```

## 颜色系统

### 主题色

| 颜色名称 | 色值 | 用途 |
|---------|------|------|
| Primary | #1976D2 | 主要操作、重要信息 |
| Secondary | #424242 | 次要操作 |
| Success | #4CAF50 | 成功状态 |
| Error | #FF5252 | 错误状态 |
| Warning | #FB8C00 | 警告状态 |
| Info | #2196F3 | 提示信息 |

### 中性色

| 颜色名称 | 色值 | 用途 |
|---------|------|------|
| Black | #000000 | 标题、重要文本 |
| Grey-Dark | #424242 | 正文文本 |
| Grey | #757575 | 辅助文本 |
| Grey-Light | #BDBDBD | 禁用文本、边框 |
| White | #FFFFFF | 背景、反色文本 |

## 间距系统

采用 8px 基准的间距系统:

- **xs**: 4px
- **sm**: 8px
- **md**: 16px
- **lg**: 24px
- **xl**: 32px
- **xxl**: 48px

## 字体规范

### 字体大小

- **标题 1**: 32px
- **标题 2**: 24px
- **标题 3**: 20px
- **标题 4**: 18px
- **正文**: 16px
- **辅助文本**: 14px
- **小文本**: 12px

### 字体粗细

- **Regular**: 400
- **Medium**: 500
- **Bold**: 700

## 响应式断点

```javascript
{
  xs: 0,      // 手机竖屏
  sm: 600,    // 手机横屏 / 平板竖屏
  md: 960,    // 平板横屏 / 小屏电脑
  lg: 1264,   // 桌面
  xl: 1904,   // 大屏
}
```

## 动画时长

- **快速**: 100ms - 用于小元素的简单过渡
- **标准**: 200ms - 用于大多数过渡效果
- **复杂**: 300ms - 用于复杂动画或大元素

---

遵循这些设计原则，可以确保组件库的质量和一致性，提供良好的开发体验。
