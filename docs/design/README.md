## 🎯 快速开始

### 1️⃣ 第一次使用？从这里开始

```
阅读顺序：
1. jianghu-ui_design_system.md      (了解全局设计原则)
2. component_categories_guide.md    (选择相关组件的规范)
3. tailwind_implementation_guide.md (查看如何实现)
```

### 2️⃣ 正在开发组件？

```
查看步骤：
1. 在 component_categories_guide.md 中找到你的组件
2. 参考该组件的视觉规范和交互规范
3. 在 interaction_animation_spec.md 查看状态转换
4. 用 tailwind_implementation_guide.md 中的示例代码实现
5. 检查 responsive_accessibility_guide.md 的响应式和无障碍要求
```

### 3️⃣ 需要设计令牌？

```
直接打开：
→ design_tokens_reference.md

复制 CSS 变量定义到项目的 src/style/design-tokens.css
```

## 📞 常见问题

**Q: 这些文档从哪里来的？**
A: 通过分析34个现有组件，提取共通的设计特征和实现模式，再结合WCAG无障碍标准和Material Design原则编写。

**Q: 可以随意修改这些规范吗？**
A: 不建议。如需调整，应该是团队决策，并同步更新所有文档和代码。

**Q: 这些规范是强制的吗？**
A: 是的。遵循规范能确保整个组件库的一致性、可维护性和用户体验。

**Q: 更新规范时需要做什么？**
A: 
1. 更新相关规范文档
2. 更新 design-tokens.css
3. 更新 tailwind.config.js
4. 更新受影响的组件代码
5. 记录变更日志

**Q: 如何提议新的设计规范？**
A: 通过创建 Issue 或 PR，说明理由和影响范围。
