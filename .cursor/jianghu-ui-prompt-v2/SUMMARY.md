## 🎯 设计理念

### 1. AI 友好
- **结构化内容**: 统一的文档结构，易于 AI 解析
- **代码优先**: 提供即用的代码模板，减少描述性文字
- **关键信息突出**: 使用表格和列表突出重要信息

### 2. 快速集成
- **核心用法**: 每个组件都有最简单的使用示例
- **快速模板**: 提供完整的可运行代码
- **常见场景**: 覆盖最常用的使用场景

### 3. 避免错误
- **注意事项**: 明确指出常见错误
- **正确示例**: 提供正确的使用方式
- **规则强调**: 重复强调必须遵守的规则

### 4. 实用为主
- **聚焦核心**: 只介绍最常用的功能
- **省略细节**: 不追求完整性，追求实用性
- **场景导向**: 按使用场景组织内容

## 🚀 使用方式

```

@JhTable.md 创建用户管理页面
@JhForm.md 创建表单
```

#### 方式二：直接提问
```
使用 JhTable 创建一个商品管理页面

```

## 💡 最佳实践

### 新手推荐学习路径

1. **第一步**: 阅读 [QUICK_START.md](./QUICK_START.md)（5分钟）
2. **第二步**: 学习核心组件
   - [JhTable.md](./JhTable.md) - 表格

   - [JhLayout.md](./JhLayout.md) - 布局

   - CRUD 列表页

   - 批量操作
   - 字段联动
   - 分步表单

### 开发建议

1. **先查文档，后写代码**
   - 不确定用法时，先查对应组件文档
   - 参考快速模板，避免从零开始

2. **遵守规则，避免错误**
   - 必须包裹在 `<v-app>` 中
   - 使用 Vue 2 Options API
   - 服务端分页用 `request`

3. **善用 AI 助手**
   - 使用 @引用文档
   - 描述清楚需求场景
   - 提供具体的字段信息
